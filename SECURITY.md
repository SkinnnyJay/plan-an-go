# Security Policy

## Supported Versions

Security updates are provided for the current major version.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Please report security issues **privately** so we can fix them before disclosure.

1. **Do not** open a public issue for a security vulnerability.
2. Email the maintainers or open a **private** security advisory on GitHub: [Security Advisories](https://github.com/SkinnnyJay/plan-an-go/security/advisories/new).
3. Include a clear description, steps to reproduce, and impact if possible.
4. You can expect an initial response within a few days. We will work with you to confirm the issue and discuss a fix and disclosure timeline.
5. If the report is accepted, we will fix the issue and credit you in the release notes (unless you prefer to remain anonymous). If we decline, we will explain why.

Thank you for helping keep plan-an-go and its users safe.
